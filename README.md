# From SDR to Sales Engineer: B2B SaaS Pipeline & Technical Analysis
Background
As a Sales Development Representative, I work closely with pipeline generation and early-stage customer conversations. This project was created to deepen my technical skillset and transition into a Sales Engineer role by analyzing sales data and translating insights into business and technical recommendations.

Objective
Simulate how a Sales Engineer supports the sales process by:
Analyzing pipeline performance
Identifying high-value customer segments
Translating data into actionable sales and technical insights
  
Tools Used
SQL
Tableau / Excel
CRM-style dataset (simulated)

Dataset Overview
The dataset simulates a B2B SaaS outbound pipeline and includes:
Lead and company data
Sales stages (SQL, Demo, Closed Won/Lost)
Deal value (revenue)
Lead source (Cold Call, LinkedIn, Email)
Customer segments (SMB, Mid-Market, Enterprise)
Geographic regions

Key Analyses
1. Revenue by Region
Identified top-performing territories to support strategic account targeting.
2. Segment Performance
Analyzed average deal size across SMB, Mid-Market, and Enterprise segments.
3. Lead Source Conversion
Evaluated outbound channels (cold call, email, LinkedIn) to determine highest ROI.
4. Pipeline Funnel Analysis
Tracked deal progression to identify bottlenecks in the sales cycle.

Key Insights
Enterprise accounts generate the highest revenue but require longer sales cycles  
LinkedIn outreach shows stronger conversion rates compared to cold calling  
The West region produces the highest revenue, indicating strong market penetration  
A significant number of deals stall at the demo stage, highlighting the need for stronger technical validation and solution alignment  

SE Perspective
This project demonstrates how technical analysis can support sales by:
Improving lead prioritization through data insights  
Supporting Account Executives with data-driven recommendations  
Identifying where technical validation is needed in the sales cycle  
Bridging the gap between business needs and product capabilities  

Outcome
This project reflects my transition from SDR to Sales Engineer by combining:
Sales experience (pipeline, outreach, qualification)
Technical skills (SQL, data analysis)
Solution oriented thinking (insight-driven decision making)
